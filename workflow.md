# Development Workflow

## Step 1: Understand Requirement
- Identify the math concept to build
- Define learning outcome clearly

## Step 2: Break Into Modules
Each feature must include:
- Concept section
- Example section
- Practice section
- Feedback system
- Tips section

## Step 3: UI Structure
- Tab-based or section-based layout
- Mobile-first design
- Clear hierarchy

## Step 4: Logic Implementation
- Store questions in structured arrays
- Separate logic from UI
- Use reusable functions

## Step 5: Feedback System
- Immediate response (correct/incorrect)
- Explanation must always be shown

## Step 6: Testing
- Check all edge cases
- Validate answers
- Ensure UI responsiveness

## Step 7: Optimization
- Remove unnecessary code
- Improve readability
- Maintain consistency

## Step 8: Update Progress
- Document completed feature in progress.md

## Coding Rules
- Keep functions small and reusable
- Avoid duplication
- Use meaningful variable names
- No unnecessary complexity

## UI Rules
- Clean layout
- Consistent spacing
- Readable fonts
- Minimal color palette

## What to Avoid
- Over-engineering
- Unnecessary animations
- Complex frameworks (early stage)
- Feature creep

## Deployment (Later Phase)
- Host on Netlify / Vercel
- Ensure fast load time
